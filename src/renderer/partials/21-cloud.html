From the R2 overview page in your Cloudflare dashboard. Used to build the endpoint https://<account-id>.r2.cloudflarestorage.com.
If the bucket has public access or a custom domain attached, beat links use this directly. Otherwise a temporary signed URL is generated.
All uploads and listings are scoped under this prefix inside the bucket.

How this works

  • Cloudflare R2 is S3-compatible object storage — it uses the same API as Amazon S3, just pointed at a different endpoint.
  • R2 has zero egress fees, so serving beat files to customers doesn't rack up bandwidth charges.
  • Beats uploaded here live in your own Cloudflare account, fully separate from local files on this PC.
  • Credentials are saved locally to this app's settings folder — never committed to source control.
